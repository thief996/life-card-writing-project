# 視覺與 IG 規格 V2.0 正式版

## 核心

**圖卡是鉤子。故事才是靈魂。**

卡面負責讓人停下來，完整故事與意義留在貼文、輪播及圖鑑頁面。

## 固定規範

- 尺寸：1080 x 1350，比例 4:5。
- 輸出：無白邊。
- 風格：手繪冒險圖鑑風。
- 插畫面積：卡面 70% 以上。
- 文字精簡：一張卡只留一句核心台詞。
- 必要資訊：名稱、稀有度、類型、能力。
- 正式系列不以方形卡作為封面。

## 系列色彩

- 主色：偏藍湖水藍，建議以 `#4B8F8C` 附近為識別基準，避免顯綠。
- 輔助色：暖米白、牛皮紙棕、墨灰。
- 主色與輔助色是系列語彙，不是每張卡必須全部使用的四色配額。
- 可依故事情緒加入其他色彩；只要仍能辨識為同一套圖鑑即可。

## 自由發揮

- 手繪媒材可依故事選擇水彩、色鉛筆、粉彩、墨線、蠟筆或混合媒材。
- 紙質不限定舊紙，可使用乾淨紙張、纖維紙、水彩紙或符合故事的表面質感。
- 邊框、構圖、光線與裝飾不必每張相同。
- 系列感主要來自固定尺寸、敘事型手繪、冒險圖鑑語彙與必要卡牌資訊。

## 製卡流程

1. 先讀懂故事，辨識真正的轉折與成長。
2. 先讓主插畫自由生長，不用大量文字綁住畫面。
3. 確認人物、場景、情緒、象徵物與故事是否準確。
4. 生成階段將人物、標題、能力與任務提示放在中央 4:5 安全區，避免後製時被裁切。
5. 再鎖定 1080 x 1350 卡面，加入名稱、稀有度、類型、能力與一句核心台詞。
6. 以重新構圖、延伸背景或安全裁切完成原生 4:5 畫面，不用左右色塊、白邊或裝飾留邊補足比例。
7. 輸出後讀取檔案實際像素；只有寬 1080、高 1350 才能標示為正式圖。
8. 檢查人物、標題、卡號、稀有度、能力與任務牌均未遭裁切或變形。
9. 原始插畫、候選版本與正式牌面分開保存。

## 尺寸鎖定與交付檢查

- 繪圖模型的尺寸指令只視為構圖目標，不視為實際輸出保證。
- 原始生成圖即使比例接近 4:5，只要像素不是 1080 x 1350，就仍是原始圖或候選圖。
- 正式圖不得以拉伸變形方式改尺寸。
- 正式圖不得以左右色塊、上下留白或額外邊框假裝符合 4:5。
- 若安全裁切會傷到人物或文字，必須重新生成或重新構圖。
- 每次交付需先驗證並回報實際像素尺寸。
- Notion「圖檔」欄位、頁面首圖與社群發布皆使用驗證完成的正式圖，不使用原始生成圖。

## Notion 圖檔巡檢規則

- 每日自動巡檢的正式圖片來源只認 `06_素材庫/正式輸出`。
- `06_素材庫/本機正式圖片`、`06_素材庫/待確認`、生成暫存資料夾與其他位置只作為人工查證線索，不作為自動上傳來源。
- 巡檢時以 `06_素材庫/正式輸出` 的檔名對照 Notion 牌卡資料庫中的卡牌名稱、卡號與卡牌類型。
- 若檔名只有尾端序號不同，例如 `卡名.png`、`卡名 (1).png`、`卡名 (2).png`，視為同一張牌卡的圖片群組。
- 圖片群組只建立一個 Notion 頁面，頁面名稱與資料庫屬性使用不含尾端序號的基礎檔名。
- 圖片群組依序放進同一頁：若有無序號原檔，先放原檔，再放 `(1)`、`(2)`、`(3)`；若沒有無序號原檔，直接從 `(1)`、`(2)`、`(3)` 依序放入。
- 只有名稱、卡號、類型與版本能明確一一對應，且圖片檔可正常讀取、不是 0 bytes，才可補進 Notion「圖檔」欄位及頁面首圖。
- 若同一卡號但基礎檔名不同，仍視為多版本或命名衝突；檔案為 0 bytes、尺寸不合規、名稱衝突、卡號衝突或分類有疑義時，一律列入待確認，不自動上傳。
- 候選圖不得覆蓋正式圖；正式輸出資料夾沒有的圖，不視為可自動同步圖。

## 輪播結構

1. 封面卡牌：系列、卡名、主插圖、卡號、稀有度。
2. 金句：一句讓人停下來的話。
3. 故事開始：具體時間、地點與處境。
4. 故事轉折：真正重要的事件。
5. 意義：當年的自己獲得了什麼。
6. 能力頁：能力值、卡牌效果與互動問題。

## 設計原則

- 卡牌是卡牌，故事是故事。
- 不把標題、完整故事、小漫畫、能力值和效果全部塞在同一張。
- 優先保留具體細節，而不是塞滿裝飾。
- 同一張卡的封面與輪播故事需維持相同的角色、物件與敘事設定，色彩可依頁面功能調整。
