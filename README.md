# 人生打怪圖鑑

把真實人生故事轉化成可收藏、可回看、可持續擴充的手繪冒險卡牌圖鑑。

這個專案同時是寫作系統、卡牌世界觀、IG 發布素材庫、Notion 圖鑑資料庫與 AI 交接文件庫。核心原則是：

> 圖卡是鉤子，故事才是靈魂。

## 專案狀態

- GitHub repository：<https://github.com/thief996/life-card-writing-project>
- 本機工作資料夾：`C:\Users\User\OneDrive\文件\.Codex\人生打怪圖鑑`
- 預設分支：`main`
- 專案型態：內容文件、圖片素材、Notion 匯入資料與 AI 交接文件。
- 目前沒有網站程式、後端服務、套件安裝或 build step。

## 專案用途

- 將日常故事、回憶、人物、物件與成長轉化為卡牌收藏。
- 管理「每日副本」流程：引路角色卡、今日任務、故事、人生卡牌、成就卡與圖鑑收錄。
- 統一保存品牌世界觀、視覺規格、牌卡資料、貼文草稿、正式圖檔與 Notion 匯入規格。
- 讓下一位 AI 或協作者能快速接手，不必從聊天紀錄重新拼湊規則。

## 目前功能

- 已建立本機資料夾結構：
  - `00_專案總覽`：專案地圖、每日副本流程、Make 自動發布藍圖。
  - `01_品牌世界觀`：品牌核心與遊戲化世界觀。
  - `02_視覺設計`：IG 卡牌視覺規格、尺寸規則、生成版本紀錄。
  - `03_牌卡資料庫`：正式牌卡、早期概念、欄位說明。
  - `04_內容系列`：四大系列與季節限定卡池。
  - `05_貼文草稿`：待發布與已完成貼文。
  - `06_素材庫`：正式輸出、待確認與對話盤點候選圖片。
  - `07_會議與對話紀錄`：重要討論與決策紀錄。
  - `08_Notion匯入`：Notion 資料庫規格與 CSV。
- 已定義每日副本流程：
  1. `GM！開啟今日副本！`
  2. 生成 1080 x 1350 無白邊引路角色卡。
  3. 發布今日任務。
  4. 使用者寫故事。
  5. 從故事誕生人生卡牌。
  6. 從成長誕生成就卡。
  7. 收錄本機與 Notion 圖鑑。
- 已定義正式圖檔規則：
  - 正式圖片來源只認 `06_素材庫/正式輸出`。
  - 圖片必須驗證為 1080 x 1350 才能視為正式圖。
  - 檔名只有 `(1)`、`(2)`、`(3)` 等尾端序號不同時，視為同一張卡的圖片群組。
  - 若沒有無序號原檔，直接從 `(1)` 開始依序放入同一個 Notion 頁面。
- 已建立 Notion 資料庫規格與資料來源：
  - Notion 專案首頁：<https://app.notion.com/p/37d4864afb9d81608b3cfff8c8d7acd7>
  - Notion 牌卡資料庫：<https://app.notion.com/p/35222c576c69445181283c42f7097ea2>

## 啟動方式

這不是一般需要啟動伺服器的程式專案，而是一個內容與素材管理專案。

接手時不需要啟動服務，建議依序閱讀：

1. `README.md`
2. `docs/PROGRESS.md`
3. `00_專案總覽/專案地圖.md`
4. `02_視覺設計/視覺規格.md`
5. `08_Notion匯入/Notion建置規格.md`
6. `03_牌卡資料庫/正式牌卡`
7. `06_素材庫/正式輸出`

日常工作入口：

- 新副本：使用口令 `GM！開啟今日副本！`
- 正式圖檔對照：檢查 `06_素材庫/正式輸出`
- Notion 同步：依 `02_視覺設計/視覺規格.md` 的「Notion 圖檔巡檢規則」處理。
- IG 文案：從 `05_貼文草稿` 讀取或新增。

版本控管常用流程：


## 部署方式

目前沒有網站或後端服務需要部署。

發布流程分成兩層：

- 內容發布：由 GM 整理文案與圖檔，使用者確認後發布 IG。
- 自動發布規劃：`00_專案總覽/Make自動發布藍圖.md` 已規劃 Make + Notion + OneDrive + Instagram for Business 的流程，但尚未確認已正式上線。

Make 預計部署邏輯：

1. Notion 監看核准發布項目。
2. 從 OneDrive 的正式輸出資料夾取得圖片。
3. 發布到 Instagram。
4. 回寫 IG 網址、Media ID、發布時間與錯誤訊息。

安全原則：

- 未勾選「核准發布」不得自動發文。
- 正式發布只讀取 `06_素材庫/正式輸出`。
- 不把 Webhook、API 金鑰或憑證寫入公開文件。
- GitHub 只作為版本控管與交接備份，不代表內容已自動發布到 IG 或 Notion。

## 環境變數

目前本專案沒有必填的本機環境變數。

若啟用 Make、Notion API、OneDrive 或 Instagram 自動發布，請使用外部平台的安全憑證管理，不要把下列內容提交到 Git：

- `NOTION_TOKEN`
- `MAKE_WEBHOOK_URL`
- `ONEDRIVE_CLIENT_ID`
- `ONEDRIVE_CLIENT_SECRET`
- `INSTAGRAM_ACCESS_TOKEN`
- 任何 Cookie、Session、OAuth refresh token 或私密 API key

## 已知問題

- `docs/PROGRESS.md` 是 2026-06-19 新增的交接日誌，較早進度需要從 `07_會議與對話紀錄` 追溯。
- 本機專案目前以內容文件與圖片為主，尚未建立自動測試或 CI。
- 正式圖檔數量已增加，但部分檔名有同卡號不同名稱、多版本或尾端序號群組，需要同步 Notion 時逐一判定。
- 圖像模型輸出尺寸不保證等於 1080 x 1350，每次都要驗證實際像素。
- Make 自動發布仍是藍圖，是否已連上正式 IG 發布流程需再確認。
- 目前沒有自動測試、CI 或圖片尺寸檢查腳本；正式圖檔仍需人工驗證。
- Notion 與 GitHub 不是自動雙向同步；若更新其中一邊，需手動確認另一邊是否也需要同步。
